# Team-Tasks 요건 정의

---

## 1. 기능 요건 (MVP)

| ID | 이름 | 설명 | 우선순위 |
|----|------|------|---------|
| FR-01 | 일감 추가 | 제목·담당자·마감일을 입력해 일감을 등록한다. 담당자는 기존 팀원 목록에서 선택한다. | 필수 |
| FR-02 | 일감 목록 조회 | 전체 일감을 단일 테이블로 표시한다. 담당자 버튼을 눌러 본인 일감만 필터링할 수 있다. | 필수 |
| FR-03 | 상태 변경 | 각 일감의 상태를 할 일 → 진행 중 → 완료 순으로 변경할 수 있다. 완료 → 진행 중 복원도 가능하다. | 필수 |
| FR-04 | 일감 삭제 | 일감을 목록에서 영구 삭제한다. | 필수 |
| FR-05 | 메모 입력 | 일감 등록 시 선택적으로 메모(상세 설명)를 함께 저장한다. | 선택 |

## 2. 비기능 요건

| 영역 | 요건 | 측정 기준 |
|------|------|----------|
| 성능 | 초기 페이지 로드 | Lighthouse FCP ≤ 1.5 s (로컬 환경 기준) |
| 성능 | 상태 변경 반응 | 버튼 클릭 후 UI 업데이트 ≤ 100 ms |
| 보안 | 민감정보 비저장 | localStorage에 비밀번호·토큰 등 인증 데이터를 저장하지 않는다 |
| 보안 | XSS 방지 | 사용자 입력을 React JSX로만 렌더링해 innerHTML 직접 삽입을 금지한다 |
| 가용성 | 오프라인 동작 | 네트워크 없이도 localStorage 기반으로 일감 조회·상태 변경이 가능하다 |
| 가용성 | 데이터 유지 | 브라우저를 닫았다 열어도 일감 데이터가 유지된다 (localStorage 영속) |
| 접근성 | 색상 대비 | 텍스트-배경 대비율 WCAG 2.1 AA 기준 ≥ 4.5 : 1 |
| 접근성 | 키보드 조작 | 모든 인터랙션(추가·상태 변경·삭제)을 키보드만으로 수행할 수 있다 |
