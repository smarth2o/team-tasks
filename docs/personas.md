# Team-Tasks 페르소나

> **MVP 헌장** — 단일 테이블 · 기능 5개 이하 · 1일 안에 코드로 옮길 수 있는 분량

---

| 구분 | 역할 | 기술 수준 | 주요 시나리오 |
|------|------|-----------|--------------|
| **P1 — 팀장 지현** | 5인 개발팀 리드. 매일 아침 스탠드업을 진행하고 일감을 분배한다. | 비개발자 관리직. 엑셀·노션 익숙, 전용 툴 경험 없음. | ① 오늘의 일감 목록을 열어 미완료 항목을 한눈에 확인한다. ② 새 일감을 추가하고 담당자를 지정한다. ③ 완료된 일감을 체크해 팀 진행률을 파악한다. |
| **P2 — 개발자 민준** | 풀스택 개발자. 하루에 여러 일감을 병렬로 처리한다. | 개발 도구에 능숙. 빠른 조작과 최소 클릭을 선호. | ① 자신에게 할당된 일감만 필터링해 오늘 할 일을 확인한다. ② 일감 상태를 진행 중 → 완료로 빠르게 변경한다. ③ 긴 설명이 필요한 일감에 메모를 추가한다. |
| **P3 — 인턴 수아** | 합류 2주차 인턴. 처음으로 팀 협업 툴을 사용한다. | IT 기기에 익숙하나 업무 툴 경험 적음. 직관적 UI 필요. | ① 자신의 이름을 찾아 오늘 담당 일감을 확인한다. ② 일감을 완료 처리하고 팀장에게 별도 보고 없이 상태를 공유한다. ③ 잘못 완료한 일감을 다시 진행 중으로 되돌린다. |

---

## MVP 기능 범위 (5개)

| # | 기능 | 대상 페르소나 |
|---|------|--------------|
| 1 | 일감 추가 (제목 · 담당자 · 마감일) | P1 |
| 2 | 일감 목록 조회 (전체 / 내 것만 필터) | P1, P2, P3 |
| 3 | 상태 변경 (대기 → 진행 중 → 완료) | P2, P3 |
| 4 | 일감 삭제 | P1 |
| 5 | 메모 입력 | P2 |
